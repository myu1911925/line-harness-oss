# 閃光舎 LINE Harness 運用マニュアル

## 管理画面URL

https://line-harness-oss-web-virid.vercel.app/

---

## 4つのサービスの役割

| サービス | 使う頻度 | 何をするところ |
|---------|---------|-------------|
| **LINE Harness管理画面** | 毎日 | 配信・友だち管理・全ての日常運用 |
| **Cloudflare** (dash.cloudflare.com) | 数ヶ月に1回 | サーバー・DB・シークレット管理。壊れた時だけ |
| **Vercel** (vercel.com) | ほぼ触らない | 管理画面のホスティング。放置でOK |
| **LINE Developers** (developers.line.biz) | ほぼ触らない | Webhook URL・LIFF設定変更時のみ |

**原則: 日常運用はLINE Harness管理画面だけ触ればOK**

## 管理画面のメニュー構成

### 配信系（よく使う）
- **シナリオ配信** `/scenarios` — 友だち追加→自動メッセージの設定
- **一斉配信** `/broadcasts` — 全員・タグ指定での配信（水曜配信はここ）
- **テンプレート** `/templates` — よく使うメッセージを保存
- **リマインダ** `/reminders` — セミナー前日・当日などの自動配信

### CRM系
- **友だち管理** `/friends` — 友だち一覧・タグ付け・メタデータ確認
- **個別チャット** `/chats` — 特定の友だちに直接メッセージ

### 分析系
- **流入経路** `/affiliates` — どこから友だち追加されたか
- **CV計測** `/conversions` — コンバージョン計測
- **フォーム回答** `/form-submissions` — アンケート回答一覧

### 自動化
- **オートメーション** `/automations` — キーワード自動返信・リッチメニュー反応など
- **Webhook** `/webhooks` — 外部サービス連携

## 水曜配信の作り方（カルーセル付き）

### 全体の流れ

1. Claudeにカルーセルの**JSONを作ってもらう**
2. 管理画面で**一斉配信を新規作成**
3. JSONを**貼り付けて予約配信**

---

### Step 1: Claudeにカルーセルを作ってもらう

こんなふうに頼む：

```
水曜配信用のカルーセルを作って。
商品：〇〇（商品名）
説明：〇〇（1〜2行）
URL：https://kimonosenkosha.com/products/xxx?utm_source=line&utm_medium=broadcast&utm_campaign=240423
画像：（後で差し替えるのでなし or 画像URLがあれば）
```

UTMパラメータは必ずつける（`utm_campaign` は配信日付 例: `240423`）

---

### Step 2: 管理画面で一斉配信を作成

1. 左メニュー「**一斉配信**」をクリック
2. 右上「**新規配信**」ボタンをクリック
3. フォームに入力：
   - **配信タイトル**: 例「2024/04/23 水曜配信 〇〇」（自分用メモ、友だちには見えない）
   - **メッセージ種別**: 「**Flexメッセージ**」を選択
   - **メッセージ内容**: ClaudeからもらったJSONを貼り付け
   - **配信対象**: 「全員」または特定タグ
   - **送信方法**: 「予約配信」→ 日時を指定（例：水曜 10:00）

4. 「**テスト送信**」で自分のLINEに届くか確認
5. 問題なければ「**予約する**」

## テスト配信の手順（本番送信前に必ず実施）

### 鉄則：送信前に「対象人数」を必ず確認する

送信確認ダイアログに **「対象: 0人」** と表示されていたら送信しない。
0人のまま送信ボタンを押しても誰にも届かない。

---

### テスト配信のやり方

1. **新規配信を作成**（カルーセルビルダーで内容を作る）
2. 配信対象：「**タグで絞り込み**」→「**テスト**」を選択
3. 「**下書きとして保存**」で作成
4. 作成した配信をクリックして詳細ページを開く
5. 配信設定欄に「**タグ: テスト (1人)**」と表示されることを確認
   - 0人と表示される場合は送信しない（Claudeに相談）
6. 「**この配信を送信する (1人)**」ボタンをクリック
7. 確認ダイアログで「**対象: 1人**」を確認してから「送信する」

---

### テスト後に全員配信に変更する手順

1. 詳細ページ右上「**内容を編集**」をクリック
2. 配信対象を「**全員**」に変更
3. 配信タイミングを「**予約配信**」→ 日時を指定
4. 「**保存**」

## UTMパラメータのルール

LINE経由のアクセスは必ずUTMをつける：

| パラメータ | 値 | 説明 |
|-----------|---|------|
| `utm_source` | `line` | 固定 |
| `utm_medium` | `broadcast` / `message` / `richmenu` | 配信種別 |
| `utm_campaign` | `240423`（日付）など | 何の配信か |

例：
```
https://kimonosenkosha.com/products/xxx
  ?utm_source=line
  &utm_medium=broadcast
  &utm_campaign=240423
```
